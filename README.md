### Step 2 вЂ” Backend setup

```bash

# Install dependencies
```

```bash

curl -X POST "http://localhost:8000/api/scrape-data"
```

This creates `data/heritage_sites.json`.

Run the NLP enrichment pipeline:

```bash

python scripts/nlp_pipeline.py
```

## рџ•ёпёЏ Knowledge Graph Schema

### Node Types

| Node | Properties |
|---|---|
| `Site` | name, id, year, description, country, image_url |
| `State` | name |
| `Category` | name (Cultural / Natural / Mixed) |
| `Keyword` | name |
| `Location` | name (NLP extracted) |
| `Person` | name (NLP extracted) |

### Relationships

```
(Site)-[:LOCATED_IN]---------->(State)
(Site)-[:TYPE]---------------->(Category)
(Site)-[:HAS_KEYWORD]-------->(Keyword)
(Site)-[:MENTIONS_LOCATION]->(Location)
(Site)-[:ASSOCIATED_WITH]--->(Person)
```

### Sample Cypher Queries

```cypher
// All sites in Maharashtra
MATCH (s:Site)-[:LOCATED_IN]->(st:State {name: 'Maharashtra'})
RETURN s.name, s.year ORDER BY s.year

// Natural heritage sites
MATCH (s:Site)-[:TYPE]->(c:Category {name: 'Natural'})
RETURN s.name, s.description

// Sites with 'temple' keyword
MATCH (s:Site)-[:HAS_KEYWORD]->(k:Keyword {name: 'temple'})
RETURN s.name, s.state

// Full subgraph for Taj Mahal
MATCH (s:Site {name: 'Taj Mahal'})-[r]->(n)
RETURN s, r, n

// Sites per state (ranked)
MATCH (st:State)<-[:LOCATED_IN]-(s:Site)
RETURN st.name AS state, COUNT(s) AS total
ORDER BY total DESC

// Shared keywords between sites
MATCH (s1:Site)-[:HAS_KEYWORD]->(k)<-[:HAS_KEYWORD]-(s2:Site)
WHERE s1 <> s2
RETURN s1.name, s2.name, collect(k.name) AS shared
```

## рџ§  NLP Pipeline

The NLP pipeline (`scripts/nlp_pipeline.py` and `backend/routers/scraper.py`) runs:

1. **Named Entity Recognition** (spaCy `en_core_web_sm`)
   - `GPE` / `LOC` в†’ locations
   - `PERSON` в†’ historical figures
   - `ORG` в†’ dynasties, institutions
   - `DATE` в†’ historical dates

2. **Rule-based extraction**
   - Dynasty patterns (Mughal, Gupta, Chola, etc.)
   - BCE/CE year patterns
   - Architecture style detection

3. **Domain keyword extraction**
   - Scans tokens against heritage vocabulary
   - Noun chunk analysis for compound terms

4. **Intent parsing** (`/api/nlp/parse`)
   - Detects: filter, count, oldest, recent, detail, general
   - Generates corresponding Cypher query
   - Executes against JSON or Neo4j
