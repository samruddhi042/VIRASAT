# 🏛️ Virasat — UNESCO Heritage India Knowledge Graph

> An AI-powered semantic exploration system for UNESCO World Heritage Sites in India.
> Built with FastAPI · React · Neo4j · spaCy · BeautifulSoup.

---

## ✨ Features

| Feature | Description |
|---|---|
| 🔍 **Semantic Search** | NLP intent parsing converts natural language to graph queries |
| 🕸️ **Knowledge Graph** | Neo4j graph with Site → State, Category, Keyword relationships |
| 🤖 **NLP Pipeline** | spaCy NER extracts locations, persons, historical references |
| 🌐 **Web Scraper** | BeautifulSoup scrapes UNESCO World Heritage pages |
| 💬 **Chat Interface** | Rule-based + NLP-powered Q&A against the knowledge graph |
| 🗺️ **Graph Visualization** | Interactive canvas graph (Cytoscape.js-style) |
| 🎨 **Premium UI** | React + Framer Motion + Tailwind — dark, gold-accent design |

## 🚀 Quick Start (Local Development)

### Prerequisites

- Python 3.10+
- Node.js 18+
- Neo4j 5 (optional — app works without it using JSON fallback)

---

### Step 1 — Clone & set up environment

```bash
git clone https://github.com/yourname/virasat.git
cd virasat

cp .env.example .env
```

---

### Step 2 — Backend setup

```bash
cd backend

# Create virtual environment
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt

# Download spaCy language model
python -m spacy download en_core_web_sm
```

---

### Step 3 — Scrape & process data

```bash
# Option A: Use the curated 42-site fallback dataset (recommended, instant)
curl -X POST "http://localhost:8000/api/scrape-data?use_fallback=true"

# Option B: Live scrape from UNESCO website (requires internet access)
curl -X POST "http://localhost:8000/api/scrape-data"
```

This creates `data/heritage_sites.json`.

Run the NLP enrichment pipeline:

```bash
cd ..
python scripts/nlp_pipeline.py
```

---

### Step 4 — Start the backend

```bash
cd backend
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

Backend available at: **http://localhost:8000**
API docs (Swagger): **http://localhost:8000/docs**

---

### Step 5 — Frontend setup

```bash
cd frontend
npm install
npm run dev
```

Frontend available at: **http://localhost:3000**

---

### Step 6 — Neo4j (optional)

Start Neo4j using Docker:

```bash
docker run -d \
  --name virasat-neo4j \
  -p 7474:7474 -p 7687:7687 \
  -e NEO4J_AUTH=neo4j/virasatpass \
  neo4j:5
```

Build the knowledge graph:

```bash
# Via API
curl -X POST http://localhost:8000/api/build-graph

# Or via script
python scripts/neo4j_setup.py
```

Neo4j Browser: **http://localhost:7474**

## 🕸️ Knowledge Graph Schema

### Node Types

| Node | Properties |
|---|---|
| `Site` | name, id, year, description, country, image_url |
| `State` | name |
| `Category` | name (Cultural / Natural / Mixed) |
| `Keyword` | name |
| `Location` | name (NLP extracted) |
| `Person` | name (NLP extracted) |

### Relationships

```
(Site)-[:LOCATED_IN]---------->(State)
(Site)-[:TYPE]---------------->(Category)
(Site)-[:HAS_KEYWORD]-------->(Keyword)
(Site)-[:MENTIONS_LOCATION]->(Location)
(Site)-[:ASSOCIATED_WITH]--->(Person)
```

### Sample Cypher Queries

```cypher
// All sites in Maharashtra
MATCH (s:Site)-[:LOCATED_IN]->(st:State {name: 'Maharashtra'})
RETURN s.name, s.year ORDER BY s.year

// Natural heritage sites
MATCH (s:Site)-[:TYPE]->(c:Category {name: 'Natural'})
RETURN s.name, s.description

// Sites with 'temple' keyword
MATCH (s:Site)-[:HAS_KEYWORD]->(k:Keyword {name: 'temple'})
RETURN s.name, s.state

// Full subgraph for Taj Mahal
MATCH (s:Site {name: 'Taj Mahal'})-[r]->(n)
RETURN s, r, n

// Sites per state (ranked)
MATCH (st:State)<-[:LOCATED_IN]-(s:Site)
RETURN st.name AS state, COUNT(s) AS total
ORDER BY total DESC

// Shared keywords between sites
MATCH (s1:Site)-[:HAS_KEYWORD]->(k)<-[:HAS_KEYWORD]-(s2:Site)
WHERE s1 <> s2
RETURN s1.name, s2.name, collect(k.name) AS shared
```

## 🧠 NLP Pipeline

The NLP pipeline (`scripts/nlp_pipeline.py` and `backend/routers/scraper.py`) runs:

1. **Named Entity Recognition** (spaCy `en_core_web_sm`)
   - `GPE` / `LOC` → locations
   - `PERSON` → historical figures
   - `ORG` → dynasties, institutions
   - `DATE` → historical dates

2. **Rule-based extraction**
   - Dynasty patterns (Mughal, Gupta, Chola, etc.)
   - BCE/CE year patterns
   - Architecture style detection

3. **Domain keyword extraction**
   - Scans tokens against heritage vocabulary
   - Noun chunk analysis for compound terms

4. **Intent parsing** (`/api/nlp/parse`)
   - Detects: filter, count, oldest, recent, detail, general
   - Generates corresponding Cypher query
   - Executes against JSON or Neo4j
